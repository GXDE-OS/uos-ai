# 小U同学|uos-ai-assistant|

## 概述

小U同学 是一款综合性的智能助手，当前版本主要提供 **AI写作**、**AI知识库**、**AI翻译** 和 **MCP&Skills** 等能力，旨在为用户提供统一的 AI 辅助体验。

在当前版本中，用户主要通过主窗口和设置窗口完成日常使用与功能配置。主窗口负责对话、助手切换和历史查看，设置窗口负责模型、知识库、MCP、随航、IM 接入和代理等内容的管理。

## 助手

### 小U同学 

小U同学 助手是当前默认的综合助手，可用于一般问答、内容改写、写作辅助和多轮追问。

运行过程中，当前版本主要有以下通用交互：

1. 在底部输入区输入问题后，可点击 **发送按钮** 发送。
2. 请求发出后，界面会进入运行态，底部 **发送按钮** 会切换为 **停止**。
3. 生成完成后，结果会保留在当前会话中，后续可继续追问。

### 运行中的交互

实际使用时，用户更常接触的是“发送后”的运行过程。根据当前版本的实际界面，当前主要有以下交互：

1. 在底部输入区输入内容后，可点击 **发送按钮** 发送；部分场景也支持回车直接发送。
2. 请求发出后，界面会进入运行态，常见状态包括思考、搜索、工具调用和回答生成。
3. 当助手仍在生成内容时，底部 **发送按钮** 会切换为 **停止**，用于中断当前任务。
4. 生成完成后，当前会话不会结束，用户可继续在底部输入区补充要求或追问。
5. 运行中的内容和完成后的结果都会保留在当前会话中，后续可通过左侧历史列表再次打开继续查看。

### AI写作

AI写作用于生成各类结构化文档。当前版本下，这一能力更适合按“写作前、写作中、写作后”三个阶段来理解。

#### 写作前

进入 **AI写作** 页面后，底部输入区会切换为带变量占位的写作模板结构。用户可以直接按模板填写身份、主题、文体、字数和内容要求，也可以直接输入自然语言需求。

![image-20260508174756636](fig/image-20260508174756636.png)

点击输入区右下侧的 **附件入口** 后，会展开补充入口。当前这里主要提供两类内容来源：

1. **参考大纲**：用于先补充章节结构，再围绕结构生成文档。
2. **本地文件**：用于上传本地资料，让写作结果参考用户提供的文件内容。

#### 写作中

当用户发送写作需求后，AI 写作会先进入大纲生成流程。若当前任务适合结构化写作，页面会先显示 **生成大纲**，然后逐步生成章节草案。

1. 发送写作需求后，页面会进入 **生成大纲** 状态。
2. 大纲生成完成后，左侧会显示分章节结构。
3. 用户可点击 **基于大纲生成文档**，继续进入正文生成流程。

在大纲编辑阶段，用户可以对章节进行调整：

1. 点击章节右上角的 **加号**，可在当前章节下新增条目。
2. 点击章节右上角的 **删除** 图标，会弹出确认框。
3. 确认框中的 **取消** 和 **删除** 都可执行。
4. 可通过拖动章节左侧拖拽手柄调整顺序。

![image-20260508174820523](fig/image-20260508174820523.png)

点击 **基于大纲生成文档** 后，页面会继续经过两个过程态：

1. **正在搜索和分析资料**
2. **正在生成文档**

#### 写作后

正文生成完成后，页面右侧会进入可编辑的文档编辑器。左侧保留大纲和生成摘要，中间会显示文档卡片。

![image-20260508174841358](fig/image-20260508174841358.png)

在该阶段，用户可以：

1. 直接在右侧正文区继续编辑内容。
2. 使用顶部工具栏进行常见文本调整。
3. 在底部输入区继续补充要求，对已生成内容做优化。
4. 通过导出菜单将当前文档另存为文件。

点击顶部工具栏或文档卡片右侧的导出入口后，会弹出导出菜单。当前可见的导出项包括：

- **另存为PDF**
- **另存为Markdown**
- **另存为Word**

![6|writing export menu](fig/p06-writing-export-menu.png)

当前实际验证中，点击 **另存为PDF** 后，文档会进入保存流程，并成功落盘到 项目周报.pdf

### AI随航

**唤醒方式**

在系统界面中选中文字后，若已开启随航功能，系统会自动显示随航工具栏；关闭后，选中文本时将不再自动显示，仍可通过设置页面重新开启。
